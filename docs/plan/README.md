# docs/plan 说明

## 这个目录是干什么的

`docs/plan/` 用来同步记录每一个开发阶段的落地计划，不是项目结束后再补的总结文档。

这个目录里的文档要跟着开发一起推进。也就是说，后续每做一个阶段，都要同步补对应计划文档，让文档和代码一起成长。

## 为什么必须维护这个目录

这个项目不是只追求“最后能跑”，还要追求：

1. 结构清晰
2. 理论能讲明白
3. 代码实现能解释
4. 验收过程可复现
5. 最终答辩时老师能快速看懂你做了什么、为什么这样做

所以 `docs/plan/` 既是开发说明，也是答辩材料的基础。

## 每篇计划文档至少要写什么

后续每个阶段文档，建议至少包含以下内容：

1. 阶段目标
2. 为什么这一阶段要现在做
3. 本阶段范围
4. 不做什么
5. 理论讲解
6. 代码结构讲解
7. 实现步骤
8. 关键函数或关键模块说明
9. 常见错误
10. 验收方式
11. 答辩可讲点

## 文档写法要求

1. 不要只列标题，必须解释清楚。
2. 不要只说“实现某功能”，要说清楚为什么这样实现。
3. 尽量让没有完整看过代码的人也能读懂。
4. 文档内容必须和当前代码状态保持一致。
5. 如果后续设计发生变化，要同步回改对应计划文档。
