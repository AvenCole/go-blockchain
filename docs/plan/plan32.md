# Plan 32：GUI 节点链控制与分布式演示

## 一、这一阶段要解决什么问题

到 Plan 31 为止，GUI 已经能看到节点：

1. 启动
2. 连接
3. 同步
4. 最近网络事件

但老师在答辩时如果想继续追问：

1. 能不能直接在 GUI 里给某个节点初始化本地链
2. 能不能直接让某个节点发交易
3. 能不能直接让某个节点挖矿并触发广播

此前还主要要靠 CLI 或控制台补充演示。  
Plan 32 的目标，就是把“节点级链控制”也直接做进 GUI。

## 二、为什么这一阶段值得做

因为课程答辩里，老师通常不会只看“节点能不能启动”，还会看：

1. 这个节点是否真的有自己的链状态
2. 这个节点是否真的能参与交易和出块
3. 你能不能把网络仿真、链状态和界面交互串成完整链路

所以这一阶段的价值在于把网络页从“观察面板”升级为“控制 + 观察一体化工作台”。

## 三、本阶段范围

### 要做的内容

1. 为 GUI 节点增加本地链初始化能力
2. 支持通过指定节点发送交易
3. 支持直接让指定节点挖矿
4. 在节点状态中显示是否已初始化、当前高度、节点 Mempool 数量
5. 在 GUI 控制台补充 `nodeinit` / `nodesend` / `nodemine` 命令
6. 同步文档

### 这一阶段不做的内容

1. 不实现完整的 RPC 服务端
2. 不做真正分布式跨主机部署
3. 不做复杂节点权限系统
4. 不把所有 CLI 指令都复制成独立 GUI 表单

## 四、核心理论讲解

### 1. 为什么节点级控制很重要

因为“主链 GUI”与“网络节点 GUI”不是同一个视角。

主链 GUI 更像单机钱包与链浏览器。  
节点级控制则更强调：

1. 某个节点本地是否已经有链
2. 某个节点是否已收到交易
3. 某个节点挖矿后是否会推动网络同步

这更接近真实区块链网络实验。

### 2. 为什么节点初始化要单独暴露

有些节点是通过连接 seed 后自动同步得到链数据；  
有些节点则需要单独作为本地起点，先初始化一条链。

如果 GUI 不暴露这个动作，答辩时就很难把“节点从空目录到可工作状态”的过程讲清楚。

### 3. 为什么节点交易与挖矿要走节点对象本身

因为节点对象已经封装了：

1. 本地链目录
2. 网络广播
3. tip 通告

所以让 GUI 直接调用节点对象的交易与挖矿能力，既能复用已有逻辑，也能保证网络广播行为不会丢。

## 六、实现步骤

### 第一步：给节点补充链状态快照能力

让节点在内部安全读取：

1. 是否已初始化
2. 当前高度
3. Mempool 数量

这样 GUI 就不用在外层反复自己拼装节点状态。

### 第二步：给节点补充本地链初始化入口

让 GUI 可以指定某个节点使用某个奖励地址创建创世块。  
如果该节点已经初始化过，则直接视为就绪状态。

### 第三步：增加节点级交易与挖矿接口

让 GUI 可以：

1. 选择一个节点
2. 指定发送方与接收方钱包
3. 通过这个节点提交交易
4. 再由这个节点挖矿

### 第四步：在 GUI 网络页加入节点链控制区

把节点：

1. 初始化链
2. 发交易
3. 挖矿

这些动作放进同一个可视化操作区。

### 第五步：让 GUI 控制台也能演示同样动作

增加：

1. `nodeinit`
2. `nodesend`
3. `nodemine`

这样答辩时既可以演示按钮操作，也可以演示命令行链路。

## 七、关键代码讲解

### 1. `network.Node.EnsureBlockchain()`

这个函数负责确保节点本地已经有链。  
它会先检查目录是否已经初始化，如果没有，再创建创世块。

这里把逻辑放进 `network.Node` 内部，而不是让 GUI 直接操作节点数据目录，目的是减少耦合。

### 2. `network.Node.ChainSnapshot()`

这个函数统一返回节点当前链状态快照：

1. `Initialized`
2. `Height`
3. `MempoolCount`

这样 GUI 网络状态展示只需要消费这个快照，不需要自己拼接多次数据库读取。

### 3. `gui.Service.SubmitNodeTransaction()`

这个函数会：

1. 从 GUI 主钱包目录加载本地钱包
2. 找到发送方钱包
3. 调用指定节点对象提交交易

这样可以复用已有钱包文件，同时让交易仍然按“节点行为”进入网络广播路径。

### 4. `gui.Service.MineNodePending()`

这个函数直接调用节点对象的挖矿能力。  
因此节点挖矿成功后，仍然会继续沿用已有的：

1. 区块广播
2. tip 通告
3. 最近网络事件记录

### 5. `frontend/src/pages/NetworkPage.tsx`

这一页新增了节点链控制区，使网络页从单纯状态展示升级为：

1. 可初始化
2. 可发交易
3. 可挖矿
4. 可观察结果

## 八、验收标准

1. GUI 可直接为某个节点初始化区块链
2. GUI 可通过某个节点发起交易
3. GUI 可直接让某个节点挖矿
4. 节点状态能显示 initialized / height / mempoolCount
5. GUI 控制台支持 `nodeinit` / `nodesend` / `nodemine`
6. 测试通过
7. 文档同步更新

## 九、常见错误

### 错误 1：节点状态只显示高度，不显示是否初始化

如果一个节点未初始化，高度值本身是不够说明问题的。

### 错误 2：GUI 直接自己访问节点数据库，不走节点封装

这样会让前端服务层和底层网络层耦合过深，也不利于后续维护。

### 错误 3：节点挖矿走另一套逻辑，不复用已有广播链路

这样会导致网络行为和 GUI 行为脱节。

### 错误 4：只有按钮，没有命令行演示入口

答辩时老师可能会要求你展示终端链路，所以控制台命令同样要补齐。
