# Plan 13：GUI 演示层

## 一、这一阶段要解决什么问题

前面的阶段已经把区块链系统的核心能力做出来了，但如果只靠命令行演示，老师很难快速把整体结构一眼看清。

Plan 13 的目标是：

1. 给项目增加一个真正可运行的桌面 GUI
2. 让 GUI 展示真实后端数据，而不是静态假页面
3. 让钱包、区块、交易池、挖矿这些能力都能在图形界面里完成演示

## 二、为什么 GUI 现在做最合适

因为到 Plan 12 为止，系统已经具备：

1. 区块链主干
2. 钱包
3. UTXO
4. 签名验证
5. UTXO 缓存
6. Merkle 树
7. PoW
8. Mempool
9. 网络模拟
10. 安全校验

这时再做 GUI，页面里展示的就都是真实功能，不会出现“UI 很多，但后端能力其实没做完”的问题。

## 三、本阶段范围

## 四、技术选择说明

### 1. 为什么选 Wails

Wails 允许继续保留 Go 后端，同时使用现代前端做桌面 GUI，非常适合课程项目：

1. 后端逻辑不需要迁移
2. 桌面演示更稳定
3. 可以直接绑定 Go 方法

### 2. 为什么前端用 React 19 + TypeScript + MUI

这一阶段页面已经不再是简单展示，而是要承载：

1. Dashboard
2. 钱包列表与创建
3. 区块浏览
4. Mempool 展示
5. 排队交易
6. 挖矿触发

TypeScript 可以减少前后端字段映射错误，MUI 可以快速搭出结构清晰的桌面界面。

### 3. 为什么还接入 React Compiler

因为你明确要求“都用最新的”，所以前端已经切到：

1. React 19
2. TypeScript / TSX
3. React Compiler

这样更符合当前 React 生态的最佳实践方向。

## 五、结构设计

### 1. 根目录只保留很薄的 Wails 入口

1. `main.go`
2. `app.go`
3. `wails.json`

这里的目标只是启动 Wails 和绑定后端接口，不把展示逻辑直接堆在入口文件中。

### 2. GUI 后端服务层放在 `internal/gui/`

已拆分为多个文件：

1. `service.go`
2. `types.go`
3. `dashboard.go`
4. `wallets.go`
5. `blocks.go`
6. `transactions.go`
7. `helpers.go`

这样比把所有 Wails 绑定逻辑都塞在一个 `app.go` 里更清晰，也更方便后续继续扩展。

### 3. GUI 使用独立数据目录

为了避免和命令行 / 节点模拟同时打开同一个 Pebble 数据库产生文件锁冲突，GUI 默认使用独立数据目录：

1. CLI / 节点：`./data`
2. GUI：`./data/gui-desktop`

如果需要覆盖，也可以通过环境变量：

1. `GO_BLOCKCHAIN_GUI_DATA_DIR`

这样能避免“GUI 和 CLI 抢同一个数据库句柄”的问题。

### 4. 前端按页面和组件拆分

已拆为：

1. `frontend/src/api/backend.ts`
2. `frontend/src/components/StatCard.tsx`
3. `frontend/src/pages/DashboardPage.tsx`
4. `frontend/src/pages/WalletsPage.tsx`
5. `frontend/src/pages/BlocksPage.tsx`
6. `frontend/src/pages/TransactionsPage.tsx`
7. `frontend/src/types.ts`

这样页面职责和共享组件职责都比较明确，不容易形成一个巨大的耦合文件。

## 九、常见错误

### 错误 1：GUI 做成静态演示

这样就不算真正的系统演示层。

### 错误 2：把所有 GUI 逻辑堆在一个文件里

这会让后续维护变得非常困难，也不符合你要求的多文件、低耦合结构。

### 错误 3：前端和链逻辑直接耦合

正确做法是通过 `internal/gui` 服务层做桥接，而不是让页面直接拼底层逻辑。

## 十、答辩时可以怎么讲

你可以这样解释：

1. 前 12 个阶段先把真实链能力全部做通
2. Plan 13 才加 GUI，这样页面展示的都是真实能力，不是静态壳
3. 技术上采用 Wails 保留 Go 后端，同时使用 React 19 + TypeScript + MUI 做桌面界面
4. 结构上把 Wails 入口、GUI 服务层、前端页面层拆开，降低耦合
