# Plan 5：UTXO 交易模型

## 一、这一阶段要解决什么问题

Plan 3 的交易还是教学版输入输出模型，发送者只是在输入里直接写地址和金额。  
Plan 5 要把这个模型升级成更接近真实区块链的 UTXO 交易模式。

这一阶段的核心问题是：

1. 交易输入不再只写“谁转出多少”
2. 输入必须引用之前某笔交易的某个输出
3. 余额不再简单做加减，而是通过未花费输出计算
4. 发送交易时要自动选择可花费输出，并支持找零

## 二、为什么 UTXO 模型必须单独做成一个阶段

因为 UTXO 是后续很多阶段的基础：

1. 签名要证明你有权花费某个输出
2. 钱包余额要来自 UTXO 扫描
3. 双花检测要基于 UTXO 是否已被引用
4. UTXO 缓存阶段要优化的对象就是这套数据

如果把它和签名、钱包、公钥哈希、手续费全揉在一起，复杂度会非常高。

所以 Plan 5 只专注于：

1. 把交易结构升级为 UTXO 风格
2. 把余额逻辑升级为 UTXO 扫描
3. 把发送逻辑升级为挑选可花费输出并自动找零

## 四、核心理论

### 1. 什么是 UTXO

UTXO 是 Unspent Transaction Output，也就是“未花费交易输出”。

可以把它理解成：

1. 一笔交易会产生若干输出
2. 只要某个输出还没有被后续交易引用，它就是未花费输出
3. 钱包余额就是自己拥有的所有未花费输出之和

### 2. 为什么余额不能一直用简单加减法

简单加减法只适合最早的教学版交易模型。  
真实区块链不会直接记录某个地址“当前余额是多少”，而是通过扫描未花费输出得出余额。

### 3. 什么是找零

如果你有一个 50 的未花费输出，想转给别人 20，那么不能只花 20 的一部分。

正确做法是：

1. 整个 50 输出被花掉
2. 创建一个 20 的输出给对方
3. 再创建一个 30 的输出回给自己

这个回给自己的输出，就是找零。

## 六、推荐数据结构升级方向

### 1. `TXInput`

当前阶段应升级为至少包含：

1. 被引用交易 ID
2. 被引用输出下标
3. 当前发送者地址

### 2. `TXOutput`

当前阶段应至少包含：

1. 输出金额
2. 收款地址

### 3. `Transaction`

仍然保留：

1. `ID`
2. `Inputs`
3. `Outputs`

但其语义已经从教学版“地址 + 金额”变成了更真实的“输入花费历史输出，输出生成新所有权”。

## 七、开发落地顺序

### 第一步：升级输入输出结构

先把 `TXInput` 和 `TXOutput` 的意义切换到 UTXO 风格。

### 第二步：实现 UTXO 扫描

从链尾向前扫描，找出某地址还没有被花费的输出。

### 第三步：实现可花费输出选择

找到足够覆盖转账金额的输出集合。

### 第四步：实现找零

当输入总额大于转账金额时，自动创建一个回给自己的输出。

### 第五步：更新发送逻辑

让 `send` 不再只是简单写一个“from -> to -> amount”的教学版交易，而是真正引用历史输出。

### 第六步：更新余额查询

让 `getbalance` 按 UTXO 汇总余额，而不是按输入输出简单加减。

### 第七步：更新打印链逻辑

让 `printchain` 打印出输入引用的交易 ID 和输出下标。

## 八、CLI 如何演示

建议演示顺序：

1. `createblockchain <miner-address>`
2. `getbalance <miner-address>`
3. `send <miner-address> <alice-address> 20`
4. `getbalance <miner-address>`
5. `getbalance <alice-address>`
6. `printchain`

这样老师能看到：

1. 创世奖励先进入矿工地址
2. 转账之后 miner 的余额从 50 变成 30
3. alice 收到 20
4. 链上交易输入已经开始引用历史输出

## 十、常见错误

### 错误 1：余额还按旧的简单加减算

这样就不再是 UTXO 模型了。

### 错误 2：没有找零

这会导致余额计算不正确，也无法向真实区块链逻辑过渡。

### 错误 3：输入不记录被引用交易和输出下标

如果没有这两个字段，就无法表达“花费的是哪一个历史输出”。

## 十一、答辩时可以怎么讲

你可以这样解释：

1. 这一阶段把交易从“教学版收支模型”升级到了“未花费输出模型”
2. 余额不再直接记录，而是从链上的未花费输出推导得出
3. 这一步是签名验证、双花检测、UTXO 缓存等后续阶段的基础
