# 当前答辩基线验证记录

本文档用于记录当前仓库在 Plan 40 完成后的封板验证路径。

## 一、目标

本次验证的目标不是继续扩功能，而是确认当前版本已经满足：

1. CLI 主链路可用
2. Go 后端测试通过
3. 前端类型检查通过
4. 前端生产构建通过
5. Wails 桌面应用可完成正式构建

以下命令已在当前仓库根目录 / 前端目录执行通过：

### Go / 后端

```bash
go test ./...
go build ./...
```

### Frontend

```bash
cd frontend
bunx tsc --noEmit
bun run build
```

### Wails

```bash
wails build -clean
```

## 四、建议的答辩前手动 smoke

在可交互桌面环境中，建议至少手动确认一次以下链路：

### 1. 钱包与主链基础

1. 打开 GUI
2. 创建钱包
3. 初始化链
4. 发送普通交易
5. 挖矿并检查新区块

### 2. 网络页工具链路

1. 启动本地节点
2. 初始化节点链
3. 通过节点发送交易
4. 触发节点挖矿
5. 确认节点焦点面板与事件区同步刷新

### 3. 网络流程长操作

至少跑一次：

1. 快速同步流程
2. 或重组流程
3. 或分区 / 合流流程

需要确认：

1. 流程状态卡会显示阶段推进
2. 流程执行期间按钮被锁定
3. 完成后能看到成功摘要
4. 若失败，能看到错误阶段和错误信息

### 4. 控制台

1. 使用命令预设填入命令
2. 直接执行预设命令
3. 检查最近命令复用是否正常

## 五、当前封板建议

如果以上自动化验证全部通过，并完成至少一次桌面 smoke，则当前版本已经适合作为：

1. 课程大作业答辩版本
2. 阶段封板版本
3. 当前主线稳定基线

当前建议：

1. 暂停新增功能
2. 只处理阻塞答辩的问题
3. 把重心转向演示顺序、讲解话术和现场环境准备
